# 채용 공고 회사 이메일 수집기

사람인(Saramin) + 잡코리아(JobKorea)에서 경영지원 관련 구인 공고를 검색하고, 해당 회사의 **공식 홈페이지에 공개된 이메일**을 자동 수집하는 도구입니다.

## 수집 흐름

```
사람인/잡코리아 공고 검색 → 회사명 추출 → 기업 페이지에서 대표자명/홈페이지 수집
→ 회사 공식 사이트에서 공개 이메일 수집 → CSV 출력
```

## 주요 기능

- **2개 사이트** 동시 검색: 사람인 + 잡코리아
- **8개 키워드** 동시 검색: 경영지원, 경리, 총무, 비서, 과제비 관리, 정부지원사업 관리, 연구비 관리, 인사
- **최근 1주일** 공고만 필터링
- **깊이 탐색**: 메인 페이지, contact/about 등 고정 경로, 문의하기 폼 링크, footer 영역까지 이메일 탐색
- **공고 ID 기준 중복 체크**: 같은 공고는 다시 수집하지 않되, 같은 회사의 새 공고는 수집
- **회사 쿨다운**: 같은 회사가 CSV에 다시 포함되려면 30일이 지나야 함 (중복 메일 방지)
- **쓰레기 필터링**: 호스팅 이메일, sentry, placeholder 등 자동 제외
- **이메일 없는 회사 제외**: CSV에는 이메일 수집 성공한 회사만 포함

## 설치 & 실행

```bash
# 1. 클론
git clone <repo-url>
cd saramin_scraper

# 2. 의존성 설치
pip install -r requirements.txt

# 3. 실행
python scraper.py
```

## 출력

- CSV 파일: `output/recruit_YYYYMMDD_HHMM.csv`
- 컬럼: `기업명`, `대표자명`, `공고 키워드`, `이메일`, `홈페이지`, `출처`

## 중복 관리

**CSV 기반 (Git 공유):**
- `output/` 폴더의 기존 CSV 파일을 읽어서 이미 수집된 회사를 자동 스킵
- 같은 회사가 CSV에 다시 포함되려면 **30일** 쿨다운 필요
- 팀원 모두 같은 이력을 공유하려면 실행 후 반드시 CSV를 commit & push

**로컬 DB (개인 캐시):**
- `scraper_history.db`는 `.gitignore`에 포함 (로컬 전용)
- 공고 ID 기준 중복 체크용 캐시 역할

### 사용 흐름

```bash
git pull                  # 최신 CSV 이력 받기
python scraper.py         # 실행
git add output/           # 결과 CSV 스테이징
git commit -m "수집 결과 추가"
git push                  # 팀과 공유
```

## 설정 변경

`scraper.py` 상단에서 수정 가능:

```python
# 검색 키워드
KEYWORDS = ["경영지원", "경리", "총무", ...]

# 같은 회사 재포함 간격 (일)
COMPANY_COOLDOWN_DAYS = 30
```

## 주의사항

- 사람인/잡코리아 이용약관에 따라 과도한 크롤링은 제한될 수 있습니다
- 수집된 이메일로 광고성 메일 발송 시 정보통신망법의 수신거부(opt-out) 요건을 준수해야 합니다
- 회사 공식 홈페이지에 공개된 이메일만 수집하며, 개인 이메일은 수집하지 않습니다
